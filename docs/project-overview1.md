### 问题定义

在复杂的大语言模型（LLM）推理任务（如数学证明、代码生成）中，模型通常仅能获得稀疏的最终结果奖励（例如 答案对/错，即 ORM）。这导致采用标准 PPO 的策略引擎在冷启动时极难收敛。本框架解决的核心问题是：

> **如何利用条件扩散模型，根据大模型推理过程中的局部中间隐状态自动生成稠密的步骤级奖励序列，从而有效训练 LLM 的 PPO 策略，并通过种群协同进化持续优化该奖励函数，最终提升复杂任务的推理准确率？**

**形式化描述：**
- 任务表现为生成长度为 $T$ 的推理轨迹，大模型在每个步骤依据上下文输出动作空间 $\mathcal{A}$ 中的 token。
- 从 LLM 的推理仿真中收集条件信息 $\tau \in \mathbb{R}^{T \times d}$，其中 $\tau$ 为该推理轨迹对应的中间隐状态序列。
- 扩散模型 $P_\theta(R|\tau)$ 依据隐状态生成连续的过程奖励序列 $R = (r_1, r_2, \dots, r_T)$。
- 使用 $R$ 作为即时奖励训练 LLM 的 PPO 策略 $\pi$，然后以最终客观验证准确率相关的指标 $\rho$（ORM）评估该策略。
- 通过 精英选择 + 截断扩散变异 + 价值对齐 的闭环迭代，持续改进扩散模型与大模型种群。



### 形式化定义

我们将大语言模型的推理对齐问题建模为**基于动态生成奖励的马尔可夫决策过程 (MDP)**。不同于传统 RLHF 依赖固定的静态奖励模型，其核心数学形式重构如下：

$$\max_{\Pi} \mathbb{E}_{\pi \in \Pi} [\rho(\pi)]$$

$$\text{s.t.} \quad \pi = \arg\max_{\pi} \mathbb{E}_{x \sim \mathcal{D}, \tau \sim \pi} \left[ \mathbb{E}_{R \sim P_{\theta}(\cdot \mid \tau)} \left[ \sum_{t=1}^T r_t \right] - \beta \text{KL}(\pi \parallel \pi_{\text{ref}}) \right]$$

其中：
- **目标：** 让策略种群 $\Pi$ 涌现出的平均最终客观指标 $\rho(\pi)$（如复杂推理任务的最终正确率 ORM）最大化。
- **约束：** 种群内的每个大模型个体策略 $\pi$ 必须是在当前扩散模型 $P_\theta$ 分配的步骤级奖励序列下的**最优生成策略**（最大化累计过程奖励，并受 KL 散度偏离惩罚）。

Evolutionary OpenRLHF 的主流程通过 **种群协同进化 + 扩散生成过程奖励 (PRM) + PPO 策略优化** 来同时搜索合适的 $P_\theta$ 和 $\pi$，并保证 $\pi$ 相对于动态下发的中间奖励信号是局部最优的。



### 算法步骤与优化问题对应关系

回顾核心问题：我们的目标是 $\max_{\Pi} \mathbb{E}_{\pi \in \Pi} [\rho(\pi)]$，约束条件是种群中的每个策略 $\pi$ 必须满足 $\pi = \arg\max_{\pi} \mathbb{E} \left[ \sum r_t - \beta \text{KL} \right]$，其中 $r_t \sim P_\theta(\cdot|\tau)$。

主流程的每一步都与上述数学定义严密对应：

#### 步骤 1：读取数据集 (Prompts & Labels)
* **数据集是什么？**
  在实现中，它包含了 $N$ 个测试 Prompt 及其对应的标准答案（Ground Truth），是客观环境的反馈来源。
* **作用：** 提供绝对的评价基准。后续计算 $\rho$（ORM 分数 / 任务适应度）时需要依赖 Labels 进行验证。这一步是**定义外层优化目标 $\rho(\pi)$ 的客观参照物**。

#### 步骤 2：随机初始化扩散模型 $P_\theta$
* **扩散模型负责什么？**
  从 LLM 收集的中间条件信息（Hidden States，即 $\tau$）生成过程奖励序列 $R$（即约束条件中的 $r_t$）。
* **作用：** 探索的起点。后续将通过种群演化，让它逐步学会“什么样的隐状态轨迹 $\tau$ 能带来最终的高分 $\rho$”。这一步是**实现动态奖励生成器**，用于构造内层 PPO 优化约束。

#### 步骤 3：随机初始化大语言模型策略 (Qwen)，数量 $M$
* **模型矩阵是什么？**
  为种群维护 $M$ 个独立的 Qwen 权重副本。在每一代中，这 $M$ 个个体都需要各自去解答数据集中的 $N$ 个 Prompts。
* **作用：** 这些独立的 Qwen 副本就是**候选策略解 $\pi \in \Pi$**。后续将利用扩散模型下发的奖励对其进行 PPO 训练，使其满足“策略是当前奖励下的最优解”这一内层约束。

#### 步骤 4：构建初始种群基线 (Initial Population)
* **执行动作：** 利用初始弱扩散模型给出的奖励 $R$，对 $M$ 个 Qwen 进行首次 PPO 训练，并用 $N$ 个 Prompts 测出初始的 ORM 分数 $\rho$。
* **作用：** 确立种群的初始适应度。这是**第一次强制让所有候选策略 $\pi$ 去满足内层的 $\arg\max$ 约束**，并将生成的 $(\tau, R, \rho)$ 沉淀为后续进化的基因库。

#### 步骤 5：协同进化迭代（执行 $K$ 次）
* **核心循环是什么？**
  - **5.1 裁判升级：** 扩散模型学习高 $\rho$ 分数的轨迹特征。
  - **5.3 变异与对齐：** 利用**截断扩散**在奖励空间生成变异奖励 $R'$ 促使 Qwen 探索新轨迹，随后用干净的扩散模型对新轨迹重新打分并再次 PPO 训练。
  - **5.5 优胜劣汰：** 按 $\rho$ 分数仅保留 Top $M$ 的个体。
* **作用：** 这一步是**求解外层最大化问题 $\max_{\Pi} \mathbb{E}[\rho(\pi)]$ 的核心引擎**。变异机制（5.3.1）负责在广阔的奖励空间中进行探索；对齐训练（5.3.5 - 5.3.7）保证探索后的策略依然满足内层 PPO 约束；而最后的截断淘汰（5.5）则构成了向上的进化压力，确保种群的客观表现严格单调递增。
