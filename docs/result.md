# Beta 分布似然损失 rho 预测模型结果记录

## 1. 实验目标

本次实验的目标是训练一个 rho 预测模型。模型输入来自 MASDiff 训练过程中导出的 `tau` 和 `R`，输出为对应 individual 的 `rho`。

使用的数据文件为：

```text
outputs/qwen_hf_debug_tau_r_rho/iteration_tau_R_rho.csv
```

CSV 中包含三列：

```text
tau,R,rho
```

其中：

- `tau`：Qwen 生成过程中的 hidden states，当前样本形状约为 `[1, 195, 2560]`
- `R`：扩散奖励序列，当前样本形状约为 `[1, 195]`
- `rho`：MASDiff 训练流程中 ORM evaluation 得到的轨迹文本评分

模型采用 Beta 分布似然损失。模型不直接回归一个标量 rho，而是根据 `tau` 和 `R` 预测 Beta 分布参数 `alpha` 和 `beta`，再用 Beta 分布均值作为 rho 的点预测：

```text
rho_pred = alpha / (alpha + beta)
```

训练损失为真实 rho 在该 Beta 分布下的负对数似然：

```text
loss = -log Beta(rho | alpha, beta)
```

## 2. 数据划分

本次使用完整 CSV 中的 101 条记录，按照 70% / 15% / 15% 划分为训练集、验证集和测试集：

| 数据集 | 样本数 |
| --- | ---: |
| 训练集 | 70 |
| 验证集 | 15 |
| 测试集 | 16 |

训练脚本为：

```text
train_beta_rho_model.py
```

训练命令为：

```bash
python3 train_beta_rho_model.py --epochs 50 --batch-size 4 --hidden-dim 256 --output-dir outputs/beta_rho_model
```

主要训练参数如下：

| 参数 | 值 |
| --- | ---: |
| epochs | 50 |
| batch_size | 4 |
| hidden_dim | 256 |
| dropout | 0.1 |
| learning_rate | 1.0e-4 |
| seed | 42 |
| device | cuda |

## 3. 验证集结果

模型按照验证集 NLL 选择最佳 checkpoint。最佳模型出现在第 17 个 epoch。

| 指标 | 验证集结果 |
| --- | ---: |
| NLL | -0.8322 |
| MAE | 0.0703 |
| RMSE | 0.0998 |

为了判断模型是否真的学到了有效信号，我还和一个简单的均值基线进行了对比。均值基线使用训练集 rho 的平均值作为所有样本的预测值。

| 指标 | 均值基线 | Beta rho 模型 |
| --- | ---: | ---: |
| MAE | 0.0856 | 0.0703 |
| RMSE | 0.1076 | 0.0998 |

从验证集看，Beta rho 模型优于均值基线。MAE 从 0.0856 降到 0.0703，RMSE 从 0.1076 降到 0.0998。这说明 `tau` 和 `R` 中确实包含了一些可以预测 rho 的信息。

## 4. 测试集结果

最终加载第 17 个 epoch 的最佳验证模型，在测试集上的表现如下：

| 指标 | 测试集结果 |
| --- | ---: |
| NLL | -0.6353 |
| MAE | 0.0983 |
| RMSE | 0.1161 |

测试集上的均值基线为：

| 指标 | 均值基线 | Beta rho 模型 |
| --- | ---: | ---: |
| MAE | 0.0911 | 0.0983 |
| RMSE | 0.1074 | 0.1161 |

从测试集看，当前模型没有明显优于均值基线，MAE 和 RMSE 都略差。这说明模型在测试集上的泛化效果还不稳定。

## 5. 训练过程分析

从训练曲线看，模型存在比较明显的过拟合迹象。

训练集上，模型持续变好：

- 训练集 NLL 从约 -0.073 降到约 -2.195
- 训练集 MAE 从约 0.0885 降到约 0.0150

但是验证集上，NLL 在第 17 个 epoch 达到最好值之后开始明显变差。到第 50 个 epoch 时，验证集 NLL 已经上升到约 1.2202。

这说明模型后期虽然在训练集上拟合得更好，但对验证集的分布建模变差了。对于 Beta 分布似然模型来说，这通常意味着模型可能变得过于自信，预测出的 Beta 分布太窄，导致真实 rho 在验证集上的似然下降。

需要注意的是，验证集 MAE 后期并没有像 NLL 一样明显变坏，有些 epoch 的 MAE 甚至略低。这说明如果只看 rho 的点预测，模型后期不一定更差；但如果按照本实验设定的 Beta 分布似然目标来评价，最佳模型应该选择第 17 个 epoch。

## 6. 当前结论

当前 Beta 分布似然 rho 预测模型已经能够正常训练，并且在验证集上表现出一定效果，说明 `tau + R` 对 rho 有预测价值。

但是，当前实验还不能说明模型已经具备稳定泛化能力。主要原因是：

1. 样本量太少，完整 CSV 只有 101 条记录。
2. 输入维度较高，`tau` 的 hidden size 为 2560，而训练样本只有 70 条。
3. 模型容量相对当前数据规模偏大，容易记住训练集。
4. 测试集结果没有超过简单均值基线，说明泛化还不可靠。

因此，本次实验更适合作为一个原型验证：Beta 分布似然损失和 `tau/R -> rho` 的训练流程已经跑通，但还需要更多样本来判断这个方法是否真正有效。

## 7. 下一步计划

下一步准备修改 MASDiff 的配置文件，重新训练并导出更多 `tau/R/rho` 样本数据。

当前配置中：

```yaml
algorithm:
  M: 100
  N: 1
  K: 1
```

这意味着本次只生成了约 101 条可用于 rho 预测模型训练的数据。为了让 Beta rho 模型获得更稳定的训练样本，下一步计划增加 MASDiff 训练过程中导出的样本量，例如：

- 增大 `K`，让进化迭代轮数更多，从而产生更多 mutant / survivor 样本。
- 适当调整 `M`，控制每轮 individual 数量。
- 考虑增大 `N`，让每个 individual 的 rho 由更多 prompt/label 样本平均得到，降低单样本评分噪声。
- 保持或扩展 `csv_record_stages`，记录更多阶段的数据，例如 `initial`、`mutant`、`survivor`。

获取更多样本后，将重新训练 Beta rho 预测模型，并重点比较：

1. 验证集 NLL / MAE / RMSE 是否继续改善。
2. 测试集是否能够稳定超过均值基线。
3. Beta 分布似然是否仍然出现过度自信问题。
4. 是否需要降低模型容量或加入更强正则化。

预期更多样本会缓解当前过拟合问题，使测试集表现更加稳定。
